## 功能特点

### 💾 数据存储

使用 localStorage 存储：
- `memo_username` - 保存的用户名
- `memo_password` - 保存的密码
- `memo_remember` - 是否记住密码
- `memo_user` - 当前登录用户信息

### 🎯 使用方式

#### 演示模式

当前为演示模式，输入任意用户名和密码即可登录。

```
用户名: admin
密码: 123456
```

或者任意其他组合都可以登录。

#### 生产模式

要集成真实的后端认证，修改 `src/utils/auth.js` 中的 `login` 函数：

```javascript
export const login = async (username, password) => {
  const response = await fetch('/api/login', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ username, password })
  })
  
  if (!response.ok) {
    throw new Error('登录失败')
  }
  
  const data = await response.json()
  
  const userData = {
    username: data.username,
    userId: data.userId,
    token: data.token,
    loginTime: new Date().toISOString()
  }
  
  localStorage.setItem('memo_user', JSON.stringify(userData))
  return userData
}
```

### 🔒 安全建议

生产环境中：

1. **不要存储明文密码** - 即使是记住密码功能
2. **使用 Token 认证** - 后端返回 JWT Token
3. **设置 Token 过期时间** - 定期需要重新登录
4. **HTTPS 传输** - 确保数据加密传输
5. **密码强度验证** - 要求足够强的密码
6. **登录失败限制** - 防止暴力破解
